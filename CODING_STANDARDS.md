# Coding Standards

## Purpose

These standards will guide implementation once an approved technology stack is in place.

## Baseline Expectations

- Write clear, maintainable, and purpose-focused code.
- Prefer small, cohesive components and explicit interfaces.
- Document non-obvious decisions and public contracts.
- Add or update automated tests for behavior changes where applicable.
- Do not introduce secrets, credentials, or sensitive data into version control.
- Run agreed formatting, linting, type-checking, and test commands before review.

## Language-Specific Standards

Language and framework conventions will be added after the stack is selected.

## Reviews

Changes should be reviewed for correctness, security, readability, test coverage, and alignment with the approved architecture.
