# Scripts

## Purpose

Provide a controlled home for repeatable repository automation.

## Responsibilities

Hold approved commands that support validation, builds, migrations, maintenance, and developer workflows.

## What Belongs Here

Documented, reviewable automation scripts with explicit inputs, safe failure behavior, and clear output expectations.

## What Must Never Be Placed Here

Application modules, secrets, one-off destructive commands, hidden deployment credentials, generated output, or undocumented manual fixes.
