# Implementation Prompt

Implement an approved change with minimal, maintainable edits.

1. Confirm the active task, applicable architecture, data, security, and quality requirements.
2. Make only scoped changes; preserve unrelated work.
3. Do not invent requirements, business facts, or external data.
4. Validate in proportion to risk and report what was verified.
5. Update the active task and decision memory when the approved work materially changes project state.
